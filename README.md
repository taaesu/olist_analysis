# Olist 전자상거래 데이터 탐색 분석



\## 프로젝트 개요

브라질 전자상거래 플랫폼 Olist의 공개 데이터셋을 전반적으로 탐색하여 주요 인사이트를 발견하는 것을 목표로 데이터베이스를 구축하고, 주문·제품·리뷰 등 다양한 테이블을 통합적으로 분석하였습니다.

실제 데이터 분석가의 업무 프로세스를 가정하여, SQL과 Python(pandas, seaborn, scipy 등)을 이용한 EDA(탐색적 데이터 분석) → 가설 설정 및 검정 → 인사이트 도출의 흐름으로 프로젝트를 설계했습니다.



\## 데이터셋

출처: Kaggle – Brazilian E-Commerce Public Dataset by Olist

데이터 기간 : 2016년 ~ 2018년

총 약 100,000건의 주문 데이터와 9개의 테이블로 구성되어 있습니다.



주요 테이블:

* orders: 주문 ID, 구매/승인/배송 관련 날짜
* order\_items: 주문별 상품, 배송비, 판매자 정보
* customers: 고객 ID, 도시(city), 주(state)
* products: 상품 ID, 카테고리
* sellers: 판매자 정보
* order\_payments: 결제 금액, 결제 횟수
* order\_reviews: 리뷰 점수, 리뷰 텍스트
* geolocation: 위도/경도, 도시, 주(state) 정보
* product\_category\_name\_translation: 상품 카테고리명(포르투갈어 → 영어 번역 매핑)



\## 분석 과정 요약

1\. SQLite 데이터베이스 생성

&nbsp;  - 9개 CSV 파일을 SQLite로 통합 ('olist.db')

&nbsp;  - 각 테이블 간 관계(ERD) 확인 및 SQL 기반 탐색

2\. 데이터 정제 및 타입 변환

&nbsp;  - 'datetime' 컬럼 변환, 결측치 처리

3\. 탐색적 데이터 분석 (EDA)

   - 주문, 제품, 결제, 리뷰, 배송 등 테이블별 탐색

4\. 가설 검정

   - 실제 배송일과 예상 배송일의 차이에 대한 t-test 수행

5\. 시각화

&nbsp;  - 'matplotlib' / 'seaborn' / 'plotly' 으로 주요 지표 시각화

6\. 인사이트 도출

&nbsp;  - 각 분석 섹션마다 관찰된 패턴 및 시사점 정리



\## 주요 분석 내용 및 인사이트



\### 배송 분석

\- 가설 : 실제 배송일과 예상 배송일의 차이는 없다  

\- 결과 :  p-value < 0.05 → 귀무가설 기각, 실제 배송일과 예상 배송일 간 유의한 차이 존재

\- 해석 : 배송예측 시스템의 정확도에 문제가 있을 가능성이 있음

\- 평균 배송 소요: 약 12일, 배송 지연 사례 다수 존재

\- 60일 이상 이후 배송된 극단값 약 0.3% → 기록 오류 또는 시스템 문제 가능성



\### 제품·매출 분석

\- 제품 카테고리별 매출 상위 10개 비중 : 전체 매출의 63%를 차지

\- 하위 10개 카테고리는 0.09%에 불과함 → 매출 편중 심화

\- 제품 가격과 배송 시간 간의 유의한 상관관계는 없음

\- 2017년 11월 24일 일매출이 급등하는 패턴이 관찰됨 → 블랙프라이데이로 인한 거래량 상승 효과



\### 리뷰 분석

\- 리뷰 평점 분포는 5점 중심으로 치우쳐진 형태를 나타냄  

\- 배송 지연이 발생한 주문은 낮은 평점과 관련 있음  



\## 사용 기술

\- SQL: 데이터 추출 및 가공

\- Python: pandas, seaborn, matplotlib, plotly, scipy



\## 결론

Olist 데이터를 통해 실제 전자상거래 비즈니스의 운영 패턴과 문제점을 다각도로 탐색하였으며,  

데이터 기반 의사결정을 위한 인사이트를 도출하는 과정을 경험했습니다.
